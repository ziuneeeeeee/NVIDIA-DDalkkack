# 🚀 Study Helper 실행 방법

> **LangGraph 기반 멀티에이전트 학습 도우미** — 백엔드(FastAPI) + 프론트엔드(React/Vite) 구성

---

## ✅ 사전 준비

| 항목 | 요구 버전 |
|---|---|
| Python | 3.10 이상 권장 |
| Node.js | 18 이상 권장 |
| npm | Node.js 설치 시 함께 제공 |
| OpenAI API Key | 필수 |

---

## 📋 전체 실행 순서 요약

```
1. Python 패키지 설치
2. .env 파일 설정 (API 키)
3. 강의자료 PDF 인덱싱 (최초 1회)
4. 백엔드 서버 실행       ← 터미널 1
5. 프론트엔드 서버 실행   ← 터미널 2 (별도)
```

---

## 1️⃣ Python 패키지 설치

프로젝트 루트 디렉터리(`study_helper_web/`)에서 실행합니다.

```bash
pip install -r requirements.txt
```

> 처음 설치 시 `sentence-transformers`, `chromadb` 등의 패키지로 인해 시간이 다소 걸릴 수 있습니다.

---

## 2️⃣ 환경변수(.env) 설정

`.env`는 API 키가 들어있어 git에 올리지 않으므로(`.gitignore`), **압축을 풀었을 때
이 파일이 없는 게 정상**입니다. 프로젝트 루트에 `.env` 파일을 새로 만들고 본인의
OpenAI API Key를 넣으세요.

```env
OPENAI_API_KEY="sk-xxxxxx..."

# (선택) LangSmith 트레이싱 — 없어도 실행 가능
LANGCHAIN_TRACING_V2=false
```

> LangSmith를 쓰려면 `LANGCHAIN_TRACING_V2=true`로 바꾸고 `LANGCHAIN_ENDPOINT`,
> `LANGCHAIN_API_KEY`, `LANGCHAIN_PROJECT`를 추가하세요. 없어도 실행에는 문제없습니다.
> `python scripts/ingest.py`로 영상(mp4 등)을 업로드하려면 [ffmpeg](https://ffmpeg.org)가
> 시스템에 설치돼 있어야 합니다(PDF·녹음파일 업로드는 필요 없음).

---

## 3️⃣ 강의자료 인덱싱 (선택 — 방 기능만 쓸 거면 건너뛰어도 됨)

> 웹 앱은 이제 **방(Room)** 화면에서 PDF·녹음·영상을 업로드하면 방마다 자동으로
> 인덱싱하므로, 이 단계는 방 기능만 쓸 경우 필요 없습니다. CLI(`main.py`)나 예전
> 전역 인덱스를 직접 다루고 싶을 때만 아래를 실행하세요.

```bash
# 1. data/ 폴더에 PDF 파일을 복사합니다. 파일명은 rag.pdf 로 지정합니다.
copy 강의자료.pdf data\rag.pdf

# 2. 인덱싱 스크립트 실행 (ChromaDB + BM25 인덱스 생성)
python scripts/ingest.py --pdf data/rag.pdf
```

실행 후 `chroma_db/` 폴더와 `bm25_index.pkl` 파일이 생성되면 성공입니다.

---

## 4️⃣ 백엔드 서버 실행 (터미널 1)

```bash
uvicorn api:app --reload
```

- 서버 주소: **http://localhost:8000**
- `--reload` 옵션은 코드 변경 시 서버를 자동 재시작합니다.
- API 문서(Swagger UI): http://localhost:8000/docs

---

## 5️⃣ 프론트엔드 서버 실행 (터미널 2 — 별도 터미널)

```bash
cd frontend
npm install      # 최초 1회만 실행 (node_modules 생성)
npm run dev
```

- 웹 앱 주소: **http://localhost:5173**
- 백엔드 서버가 먼저 실행 중이어야 합니다.

> ✅ `node_modules/` 폴더가 이미 존재하면 `npm install`은 생략 가능합니다.

접속하면 **"방(Room)" 목록 화면**이 뜹니다. 이름을 붙여 방을 만들고, 그 방 안에서
PDF·녹음파일·녹화영상을 업로드하면(여러 개 가능) 핵심개념을 자동 추출·병합해
[단순 개념 확인]/[모의고사]를 문제 수 선택 없이 바로 생성합니다. 채점 결과는
방마다 "오답노트"에 자동 저장되어 문제·내 답안·정답·채점 근거를 나중에 다시 볼 수
있습니다.

---

## 6️⃣ (선택) CLI 모드 실행

웹 브라우저 없이 터미널에서만 사용하려면:

```bash
python main.py
```

터미널에서 개념을 입력 → 문제 생성 → 답안 입력 → 채점 순서로 진행됩니다.

## 🛠️ 자주 발생하는 오류 해결

### ❌ `ModuleNotFoundError` 발생 시
```bash
pip install -r requirements.txt
```
패키지가 누락된 경우 위 명령으로 재설치합니다.

### ❌ `OPENAI_API_KEY` 관련 오류 시
`.env` 파일의 API 키가 올바른지 확인하고, 파일이 프로젝트 루트(`study_helper_web/`)에 있는지 확인합니다.

### ❌ 포트 충돌 오류 시
다른 포트를 지정하여 실행합니다.
```bash
# 백엔드를 8001 포트로 실행
uvicorn api:app --reload --port 8001

# 프론트엔드를 다른 포트로 실행
npm run dev -- --port 5174
```

### ❌ `chroma_db` 관련 오류 시 (방 기능 사용 중이라면 보통 해당 없음)
방 화면에서 자료를 업로드했는데도 나는 오류라면 방 자체 인덱스(`rooms/<방ID>/chroma_db`)
문제일 수 있습니다. CLI(`main.py`)나 예전 전역 인덱스를 쓰는 경우라면 `scripts/ingest.py`를
다시 실행하세요.
