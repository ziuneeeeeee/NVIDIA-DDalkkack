# 수강 과목 학습 도우미 v2 — 아키텍처 명세서

> 본 문서는 기존 v1(정적 문제 뱅크 기반)에서 현재 **v2(실시간 멀티에이전트 생성·채점 시스템)**로 전환되며 이루어진 모든 설계 결정, 구조 변경, 버그 수정 내역을 기록합니다.

---

## 1. v1 → v2 주요 변경 사항

| 구분 | v1 (기존 설계) | v2 (현재 구현 완료) |
|---|---|---|
| **문제 출처** | 하드코딩된 샘플 문제 | **RAG 기반 실시간 AI 문제 생성** |
| **코드 구조** | 단일 파일(`grade.py`) 비대화 | **도메인별 계층화** (`core/`, `nodes/`, `graphs/`) |
| **채점 척도** | 유형별 점수 기준 혼재 | **전 유형 10점 만점** 통일 |
| **시스템 흐름** | 단일 파이프라인 | **[생성 그래프] → [채점 그래프]** 2-Phase 구조 |
| **RAG 활용** | 채점 근거 확인용 | **문제 출제 + 채점 근거** 양방향 활용 |
| **프론트엔드** | 없음 (CLI 전용) | **React (Vite) 웹 앱** 구현 완료 |
| **모의고사** | 없음 | **범위 기반 병렬 다문항 출제 + 100점 환산** |
| **API** | 없음 | **FastAPI 서버** (`/generate_problem`, `/grade_answer`, `/generate_mock_exam`, `/grade_mock_exam`) |

## 3. 핵심 파이프라인 상세

### Phase 1 — 문제 생성 (`generation_graph`)

```
사용자 입력 (개념, 난이도)
    ↓
[retrieve] RAG 하이브리드 검색 → 강의자료 컨텍스트 확보
    ↓
[generate_problem_node] Generator AI: 문제 초안 생성 (GPT-4o)
    ↓
[verify_problem_node] Validator AI: 팩트 체크 (GPT-4o)
    ↓
[judge_difficulty_node] Difficulty AI: 난이도 체감 평가 (GPT-4o)
    ↓
[conclude_problem_node] Concluder AI: 최종 승인 or 반려 (최대 3회 루프)
    ↓
Problem 객체 반환
```

### Phase 2 — 채점 (`grading_graph`)

```
Problem + 학생 답안 입력
    ↓
[parse_problem] 이미지 OCR 전처리 (해당 시)
    ↓
[retrieve] 채점 근거용 강의자료 재검색
    ↓
[get_or_create_rubric] 루브릭 생성 (서술형만) + 4종 검증
    ↓
┌─────────────────────────────────────────┐
│  유형 분기 (route_by_type)               │
│  객관식·단답형 → [grade_objective]       │
│  서술형       → [critique_strict]        │
│                  [critique_lenient]     │
│                  [critique_keyword]     │
│                  [judge_grading]        │
│  코딩형       → [grade_code]            │
└─────────────────────────────────────────┘
    ↓
[diagnose] 오답 원인 분석
    ↓
GradeResult 반환 (점수 + 항목별 피드백 + 진단 + 추천)
```

### Phase 3 — 종합 모의고사 (`core/exam.py`)

```
사용자 입력 (범위, 문제 수, 난이도)
    ↓
[orchestrate_exam_concepts] GPT-4o: 중복 없는 세부 개념 N개 추출
    ↓
[ThreadPoolExecutor] N개 개념 동시 병렬 생성 (Phase 1 × N)
    ↓
모든 문제 표시 → 학생 전체 답안 입력
    ↓
[ThreadPoolExecutor] N개 답안 동시 병렬 채점 (Phase 2 × N)
    ↓
각 문항 점수를 100/N점 단위로 스케일링
    ↓
[GPT-4o] 종합 성적 리포트 피드백 생성
    ↓
MockExamResult 반환 (총점 + 문항별 결과 + 피드백)
```

## 5. 프론트엔드 UI 구조 (`frontend/`)

| 모드 | 주요 화면 |
|---|---|
| **단건 학습** | 개념 입력 → 문제 출제 → 답안 작성 → 채점 결과 (점수 바, 항목별 평가, 오답 진단, 추천 학습) |
| **종합 모의고사** | 범위·문항 수 설정 → 전체 문제 목록 표시 → 일괄 제출 → 총점·문항별 결과·AI 피드백 성적표 |

**디자인 시스템**
- 폰트: **Inter** (Google Fonts)
- 배경: **오로라 블롭 4개** (보라·파란·핑크·청록) 애니메이션 + 미세 노이즈 텍스처
- 카드: **Glassmorphism** (`backdrop-filter: blur(30px)`) + 상단 글로우 라인
- 버튼: 흰색 배경 + 검정 텍스트 (고대비 primary), ghost 스타일 secondary
- 색상 팔레트: `#7c3aed` (보라) · `#60a5fa` (파랑) · `#34d399` (그린) 네온

## 6. 버그 및 안정성 개선 완료 내역

| 이슈 | 원인 | 해결 |
|---|---|---|
| 500 Internal Server Error | FastAPI에서 Pydantic 모델을 직접 반환하지 않고 수동 직렬화 | `return problem` 형태로 직접 반환 수정 |
| 채점 점수 항상 0점 | `per_criterion` 키 불일치 (`point_name` vs 다른 명칭) | 전 채점 노드 출력 포맷 `{point_name, earned_score, reason}` 통일 |
| 루브릭·피드백 영어 출력 | 프롬프트에 언어 지시 없음 | 모든 프롬프트에 "반드시 한국어로 작성" 강제 추가 |
| 10점 만점 체계 불일치 | 객관식이 1.0점 기준으로 채점됨 | 모든 유형 10점 만점으로 통일 |
| 윈도우 UnicodeEncodeError | 이모지 출력 시 cp949 에러 | `sys.stdout.reconfigure(encoding='utf-8')` 추가 |
| ModuleNotFoundError | `ingest.py` 실행 시 `core` 패키지 미인식 | 실행 경로 및 `sys.path` 보정 |
