# study_helper 파일 구조 및 서술 (v2 기준)

> 본 문서는 현재 멀티에이전트(LangGraph 기반) 문제 생성·채점·모의고사 시스템으로 구성된 **v2 아키텍처**를 기준으로, 프로젝트 내 모든 파일과 디렉터리의 역할을 상세히 설명합니다.

---

## 1. 최상위(Root) 파일

### `main.py`
- **터미널(CLI) 기반 실행 진입점**입니다.
- `run_generation(concept, difficulty)` 함수로 문제를 생성하고, `run_grading(problem, answer)` 함수로 채점을 실행합니다.
- 두 함수는 `api.py`에서도 동일하게 `import`하여 재사용됩니다.
- `_grading_cache` 딕셔너리로 동일 (문제, 답안) 조합의 재채점을 방지합니다 (SHA-256 해시 키).
- Windows 환경에서 이모지 출력 시 발생하는 `UnicodeEncodeError`를 방지하기 위해 `sys.stdout.reconfigure(encoding='utf-8')`을 적용합니다.

### `api.py`
- **FastAPI 기반 백엔드 서버**입니다. 프론트엔드(`frontend/`)와 JSON으로 통신합니다.
- CORS 미들웨어가 설정되어 있어 `localhost:5173`(Vite)에서의 요청을 허용합니다.
- **엔드포인트 목록**:
  - `GET /` — 서버 헬스체크
  - `POST /generate_problem` — 단건 문제 생성 (`concept`, `target_difficulty` 입력)
  - `POST /grade_answer` — 단건 답안 채점 (`problem`, `student_answer` 입력)
  - `POST /generate_mock_exam` — 모의고사 문제 일괄 생성 (`topic_range`, `num_questions`, `target_difficulty` 입력)
  - `POST /grade_mock_exam` — 모의고사 답안 일괄 채점 (`problems[]`, `student_answers[]` 입력)

### `test_exam.py`
- 모의고사 파이프라인(`core/exam.py`)을 백엔드 서버 없이 직접 테스트하는 디버그 스크립트입니다.

### `.env`
- `OPENAI_API_KEY` (필수) 및 LangSmith 트레이싱 관련 키를 저장합니다.
- 절대로 Git에 커밋하지 않습니다 (`.gitignore` 적용).

### `requirements.txt`
- 시스템 구동에 필요한 Python 패키지 목록을 관리합니다.
- 주요 패키지: `langgraph`, `openai`, `chromadb`, `rank-bm25`, `sentence-transformers`, `fastapi`, `uvicorn`, `pypdf`, `pydantic`, `python-dotenv`

## 2. `core/` — 공통 핵심 모듈

데이터 구조, 상태, 외부 API 연동 등 시스템 전반에서 공유하는 기반 코드입니다.

### `core/models.py`
Pydantic `BaseModel`로 시스템 전체의 데이터 규격을 정의합니다.

| 클래스 | 역할 |
|---|---|
| `RubricCriterion` | 채점 기준표 항목 1개 (`point_name`, `max_score`, `description`, `source_reference`) |
| `Problem` | 출제된 문제 1개 (`problem_id`, `type`, `question`, `answer`, `model_answer`, `test_cases`, `rubric`) |
| `CriterionResult` | 항목별 채점 결과 1개 (`point_name`, `score_earned`, `reason`) |
| `CritiqueResult` | 채점관 1명의 전체 채점 결과 (`critic`, `total_score`, `breakdown`) |
| `GradeResult` | 최종 채점 결과 (`final_score`, `max_score`, `per_criterion`, `confidence`, `needs_human_review`, `grader_agreement`) |
| `DiagnosisResult` | 오답 원인 진단 결과 (`error_type`, `weak_concept`, `detail`) |
| `ConceptMastery` | 개념별 학습 숙정도 추적 (향후 활용) |

### `core/state.py`
LangGraph 노드 간에 주고받는 상태(State) TypedDict를 정의합니다.
- `GradingState`: 채점 파이프라인 상태 (문제, 답안, 컨텍스트, 루브릭, 채점 결과 등)
- `GenerationState`: 문제 생성 파이프라인 상태 (개념, 초안, 피드백, 승인 여부 등)

### `core/clients.py`
- `get_openai_client()`: OpenAI 클라이언트를 싱글턴으로 초기화하여 반환합니다.
- LangSmith 트레이싱이 환경변수로 활성화된 경우 자동으로 래퍼를 씌웁니다.

### `core/exam.py`
종합 모의고사 전체 흐름을 담당하는 오케스트레이터 모듈입니다.

| 함수 | 역할 |
|---|---|
| `orchestrate_exam_concepts(topic_range, num_questions)` | GPT-4o Structured Output으로 중복 없는 세부 개념 N개 추출 |
| `generate_mock_exam(topic_range, num_questions, difficulty)` | `ThreadPoolExecutor`로 N개 문제 병렬 생성 |
| `grade_mock_exam(problems, student_answers)` | `ThreadPoolExecutor`로 N개 답안 병렬 채점, 100점 환산, 종합 피드백 생성 |

## 3. `graphs/` — 파이프라인 조립 계층

LangGraph `StateGraph`로 노드들을 연결해 실제 워크플로우를 조립합니다.

### `graphs/generation_graph.py`
**문제 생성 파이프라인** 구성:
```
retrieve → generate_problem → verify_problem → judge_difficulty → conclude_problem
                                    ↑__________________________________|
                                         (반려 시 최대 3회 재시도)
```

### `graphs/grading_graph.py`
**채점 파이프라인** 구성:
```
parse_problem → retrieve → get_rubric
                              ↓
            ┌─────────────────────────────────┐
            │ route_by_type                   │
            │ 객관식·단답형 → grade_objective  │
            │ 서술형 → critique_strict         │
            │          critique_lenient        │
            │          critique_keyword        │
            │          judge_grading           │
            │ 코딩형 → grade_code              │
            └─────────────────────────────────┘
                              ↓
                          diagnose → END
```

## 4. `nodes/` — 개별 AI 에이전트 및 기능 노드

### `nodes/retrieve.py`
- ChromaDB(벡터 검색)와 BM25(키워드 검색)를 결합한 **하이브리드 RAG** 검색을 수행합니다.
- `GradingState`와 `GenerationState` 모두에서 호출 가능하도록 유연하게 설계됩니다.

### `nodes/generation.py`
4명의 문제 생성 에이전트 함수가 정의됩니다.
- `generate_problem_node`: 강의자료 컨텍스트 기반 문제 초안 생성 (GPT-4o)
- `verify_problem_node`: 팩트체크 검증 (GPT-4o)
- `judge_difficulty_node`: 체감 난이도 평가 (GPT-4o)
- `conclude_problem_node`: 최종 승인/반려 판정, 승인 시 `Problem` 객체 생성

### `nodes/rubric.py`
- `generate_rubric()`: GPT-4o Structured Output으로 10점 만점 루브릭 생성
- `run_rubric_verification()`: 완전성·배타성·근거일치성·배점타당성 4종 검증 에이전트 실행 (GPT-4o-mini)
- `get_or_create_rubric()`: 캐시 조회 → 없으면 생성+검증, 서술형 이외엔 빈 리스트 반환
- 모든 출력에 "반드시 한국어로 작성" 지시를 프롬프트에 강제합니다.

### `nodes/essay_grading.py`
서술형 채점 전담 3-에이전트.
- `critique_strict()`: 강의자료 문자 그대로 대조하는 깐깐한 채점 (GPT-4o)
- `critique_lenient()`: 의미상 맞으면 인정하는 관대한 채점 (GPT-4o)
- `critique_keyword()`: 핵심 키워드 포함 여부 중심 채점 (GPT-4o-mini)
- 모든 `reason` 출력은 한국어 강제, `score_earned`는 루브릭 `max_score` 이하로 클리핑됩니다.

### `nodes/objective_grading.py`
- 정답 문자열을 정규화(`소문자 + 특수문자 제거`)하여 학생 답과 비교합니다.
- LLM 호출 없이 즉각 채점, 프론트엔드 포맷(`point_name`, `earned_score`, `reason`)에 맞춰 반환합니다.

### `nodes/code_grading.py`
- Python `subprocess`로 학생 코드를 샌드박스 환경에서 실행합니다.
- 문제의 `test_cases` 배열을 순회하며 테스트 통과율을 기반으로 10점 만점 환산 채점합니다.

### `nodes/judge.py`
- 3명 채점관의 `total_score` 중앙값을 최종 점수로 채택합니다.
- 점수 편차가 `max_score × 30%` 초과 시 `needs_human_review=True` 플래그를 세웁니다.
- 중앙값에 가장 가까운 채점관의 `breakdown`을 대표 항목별 피드백으로 선택합니다.

### `nodes/diagnose.py`
- 채점 결과를 보고 오답 원인을 `"개념 자체를 모름" | "계산 실수" | "문제 이해 오류" | "유사 개념과 혼동"` 4유형으로 진단합니다.

### `nodes/adapt.py` / `nodes/recommend.py`
- 연속 정답/오답 횟수에 따라 다음 문제 난이도를 상하향 조절합니다.
- 다음에 학습하면 좋을 관련 개념을 추천합니다.

### `nodes/parse_problem.py`
- 이미지 URL이 있는 문제는 GPT-4o Vision으로 OCR 파싱합니다.
- 현재는 플레이스홀더 수준이며, **향후 손글씨 답안지 업로드 자동 채점** 기능으로 확장될 예정입니다. (`보완방안.md` 섹션 8 참조)

## 5. `scripts/`

### `scripts/ingest.py`
- `data/rag.pdf` 경로의 강의자료 PDF를 **400토큰 청크** 단위로 분할합니다.
- 각 청크를 `sentence-transformers` 모델로 임베딩하여 **ChromaDB**(벡터 검색)에 저장합니다.
- 동일 청크를 **BM25 인덱스**(키워드 검색)에도 직렬화하여 저장합니다.
- 시스템 최초 구동 전 **1회 필수 실행**합니다.

## 6. `frontend/`

### `frontend/src/App.tsx`
- **단건 학습 모드**: 개념 입력 → 문제 출제 → 답안 작성 → 채점 결과 시각화
- **종합 모의고사 모드**: 범위·문항수 설정 → 전체 문제 표시 → 일괄 답안 제출 → 성적표
- TypeScript 타입 인터페이스: `Problem`, `GradeResult`, `GradingResponse`, `MockExamResult`
- `ScoreBar` 컴포넌트: 문항별/전체 점수를 시각적 바로 렌더링
- fetch API로 `http://localhost:8000`의 FastAPI 서버와 통신

### `frontend/src/index.css`
- **디자인 시스템 전체**를 정의하는 단일 CSS 파일입니다.
- 폰트: **Inter** (Google Fonts)
- 배경: 순수 `#000` 위에 오로라 블롭 4개 (`aurora-blob`) 느리게 부유하는 애니메이션
- 카드: `rgba(255,255,255,0.04)` 반투명 + `backdrop-filter: blur(30px)` + 상단 1px 글로우 라인
- 버튼: `btn-primary` (흰 배경·검정 글자), `btn-ghost` (투명·회색 테두리)
- 입력창: `rgba(255,255,255,0.06)` 배경, 포커스 시 보라 글로우
- 점수 숫자: `4.5rem` + 보라→파랑 그라디언트 텍스트 + slide-up 애니메이션
