# 🌐 Study Helper 배포 방법 가이드

> **서버 없이 GitHub 기반으로 무료(or 저비용)로 배포하는 방법 정리**  
> FastAPI 백엔드 + React/Vite 프론트엔드 + ChromaDB + ML 모델 구성

---

## ⚠️ 이 프로젝트 배포가 까다로운 이유

일반적인 웹앱과 달리 이 프로젝트는 아래 구성요소 때문에 단순 배포가 어렵습니다.

| 구성요소 | 문제점 |
|---|---|
| `sentence-transformers` | 모델 파일 수백 MB → RAM 많이 필요 |
| `ChromaDB` | 디스크에 데이터 저장 → 영구 스토리지 필요 |
| `bm25_index.pkl` | 재시작 시 파일 유지 or 재생성 필요 |
| `uvicorn` (FastAPI) | 상시 실행 서버 필요 |

```
    └── 프론트엔드 (React) →  Vercel 무료 플랜
```

## 🛤️ 방법 1 — Render + Vercel (데모용, 완전 무료)

### 1-1. GitHub 레포 준비

```bash
# 프로젝트 루트에서
git init
git add .
git commit -m "initial commit"
git remote add origin https://github.com/YOUR_ID/study-helper.git
git push -u origin main
```

> ⚠️ `.env` 파일은 `.gitignore`에 반드시 추가하세요 (API 키 노출 방지)

```gitignore
# .gitignore 에 추가
.env
__pycache__/
*.pyc
```

### 1-2. ChromaDB 데이터 레포에 포함 (핵심!)

재시작 시 데이터 소실 문제를 해결하는 가장 간단한 방법입니다.

```bash
# 인덱싱 후 생성된 파일들을 레포에 포함
git add chroma_db/ bm25_index.pkl
git commit -m "add rag index files"
git push
```

> ✅ 이렇게 하면 Render가 재시작되어도 코드와 함께 인덱스 파일이 배포됩니다.  
> ⚠️ 강의자료가 바뀌면 로컬에서 재인덱싱 후 다시 push하면 됩니다.

### 1-3. Render — 백엔드 배포

1. [https://render.com](https://render.com) 가입 (GitHub 계정 연동)
2. **New → Web Service** 선택
3. GitHub 레포 연결
4. 아래와 같이 설정:

| 항목 | 값 |
|---|---|
| **Environment** | `Python 3` |
| **Build Command** | `pip install -r requirements.txt` |
| **Start Command** | `uvicorn api:app --host 0.0.0.0 --port $PORT` |
| **Plan** | `Free` |

5. **Environment Variables** 탭에서 API 키 입력:

```
OPENAI_API_KEY      = sk-xxxxxx...
LANGCHAIN_TRACING_V2 = false
```

6. **Deploy** 클릭 → 완료 시 `https://your-app.onrender.com` 주소 발급

> ⚠️ **슬립 주의**: 무료 플랜은 15분 미사용 시 서버가 잠들며, 첫 요청에 30~60초 소요됩니다.

### 1-4. Vercel — 프론트엔드 배포

1. [https://vercel.com](https://vercel.com) 가입 (GitHub 계정 연동)
2. **New Project → Import Git Repository** 에서 레포 선택
3. 아래와 같이 설정:

| 항목 | 값 |
|---|---|
| **Framework Preset** | `Vite` |
| **Root Directory** | `frontend` |
| **Build Command** | `npm run build` |
| **Output Directory** | `dist` |

4. **Environment Variables** 추가:

```
VITE_API_BASE_URL = https://your-app.onrender.com
```

5. **Deploy** 클릭 → `https://your-app.vercel.app` 주소 발급

### 1-5. 프론트엔드 API 주소 수정

`frontend/src/` 의 API 호출 부분에서 백엔드 주소를 환경변수로 변경합니다.

```typescript
// 수정 전 (로컬)
const API_BASE = "http://localhost:8000";

// 수정 후 (배포용)
const API_BASE = import.meta.env.VITE_API_BASE_URL || "http://localhost:8000";
```

## 🚀 방법 2 — Google Cloud Run + Vercel (실 서비스, 안정적)

### 특징
- 요청이 없을 때는 비용 0원 (서버리스)
- 월 200만 요청까지 무료
- ChromaDB 데이터를 Google Cloud Storage에 저장 → 영구 보존
- Docker 컨테이너 기반

### 2-1. Dockerfile 생성

프로젝트 루트에 `Dockerfile`을 생성합니다.

```dockerfile
FROM python:3.11-slim

WORKDIR /app

# 시스템 패키지 설치
RUN apt-get update && apt-get install -y \
    gcc \
    && rm -rf /var/lib/apt/lists/*

# Python 패키지 설치
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# 소스 코드 복사
COPY . .

# 포트 설정
EXPOSE 8080

# 서버 실행
CMD ["uvicorn", "api:app", "--host", "0.0.0.0", "--port", "8080"]
```

### 2-2. Google Cloud 설정

```bash
# Google Cloud CLI 설치 후
gcloud auth login
gcloud config set project YOUR_PROJECT_ID

# Cloud Run에 배포
gcloud run deploy study-helper-backend \
  --source . \
  --platform managed \
  --region asia-northeast3 \
  --allow-unauthenticated \
  --memory 2Gi \
  --set-env-vars OPENAI_API_KEY=sk-xxxxx
```

> 완료 시 `https://study-helper-backend-xxxxxx-an.a.run.app` 형태의 주소 발급

### 2-3. ChromaDB 영구 저장 (Cloud Storage 연동)

ChromaDB를 Google Cloud Storage와 연동하려면 `core/clients.py` 등에서 ChromaDB 경로를 환경변수로 제어합니다.

```python
import os
chroma_path = os.getenv("CHROMA_DB_PATH", "./chroma_db")
```

## 🔄 방법 3 — Railway (간단하고 안정적, 월 $5 크레딧)

Railway는 설정이 가장 간단하며 슬립 없이 상시 실행됩니다.


```
OPENAI_API_KEY = sk-xxxxx...
```

## 🧠 ChromaDB 영구 저장 근본 해결 — Pinecone으로 교체

무료 환경에서 ChromaDB 데이터 소실 문제를 근본적으로 해결하는 방법입니다.

### Pinecone 무료 플랜
- 벡터 1개 인덱스, 2GB 저장 무료
- 클라우드에 데이터 저장 → 서버 재시작과 무관

### 교체 방법 (개략)

```bash
pip install pinecone-client
```

```python
# nodes/retrieve.py 수정
import pinecone

pinecone.init(api_key="YOUR_PINECONE_API_KEY", environment="gcp-starter")
index = pinecone.Index("study-helper")

# chromadb 대신 pinecone으로 벡터 검색
```

> `.env`에 `PINECONE_API_KEY` 추가 필요
